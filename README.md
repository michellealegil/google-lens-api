# Google Lens Exact Match API

A scalable API that accepts an image URL, performs a Google Lens visual search, and returns the full HTML of the Exact Match results page.

---

## Approach

### How it works

The API uses a **hybrid approach** — direct HTTP to reverse-engineer the URL, then a pooled stealth browser to render the results:

**Step 1 — Reverse-engineer the Lens URL (httpx, no browser)**

Rather than navigating the Google Lens UI, we go directly to:
```
https://lens.google.com/uploadbyurl?url={encoded_image_url}
```
Google processes the image and redirects to a `google.com/search` URL containing a dynamically-generated `vsrid` session token and all required parameters. We follow this redirect with `httpx` and extract the direct search URL — skipping all UI interaction entirely.

**Step 2 — Render the Exact Match page (Playwright browser pool)**

We append `udm=26` to the search URL (Google's parameter for the Exact Match tab) and load it in a pooled stealth browser. JavaScript executes, the results render, and we return the full HTML.

This is significantly faster than full UI navigation (no clicking through the Lens interface) and produces complete, JS-rendered results.

### Why a browser is still needed

During development, we investigated hitting Google's internal async results endpoint (`/async/folsrch`) directly with `httpx` — the true pure-API approach. This endpoint requires session parameters (`ei`, `mlro`, `mrc`) that are generated by JavaScript after the initial page load and are not present in the raw HTML response. Without executing that JavaScript, these parameters cannot be obtained, making a fully browserless approach infeasible for Google Lens Exact Match results.

### Anti-bot evasion

- **`playwright-stealth`** patches the browser to remove automation signals (`navigator.webdriver`, etc.)
- **Realistic Chrome headers** — full header set matching a real Chrome 124 browser on macOS
- **Random delays** between requests to avoid pattern detection
- **Browser pool** — reuses existing browser contexts rather than launching fresh browsers per request, reducing fingerprint churn
- **Proxy rotation** — rotates through a configurable list of proxies (see Configuration)

## Setup

### Requirements

- Python 3.9+
- Node.js (not required — pure Python)

### Install dependencies

```bash
pip3 install -r requirements.txt
python3 -m playwright install chromium
```

### requirements.txt

```
fastapi
uvicorn[standard]
httpx
playwright
playwright-stealth
```

---

## Running locally

```bash
python3 -m uvicorn main:app --reload --port 8000
```

The server starts a pool of 3 browser contexts on startup. You'll see:
```
[pool] Started with 3 browser contexts
INFO: Application startup complete.
```

### With proxies

Pass a comma-separated list of proxy URLs via the `PROXY_LIST` environment variable:

```bash
PROXY_LIST="http://user:pass@host1:port,http://user:pass@host2:port" \
python3 -m uvicorn main:app --port 8000
```

### Configuration

| Environment Variable | Default | Description |
|---|---|---|
| `PROXY_LIST` | (none) | Comma-separated proxy URLs |
| `POOL_SIZE` | `3` | Number of browser contexts to keep alive |

## API Reference

### `GET /google-lens`

Returns the raw HTML of the Google Lens Exact Match results page for a given image URL.

**Query Parameters**

| Parameter | Required | Description |
|---|---|---|
| `imageUrl` | Yes | The image URL to search |

**Response**

- `200 OK` — HTML string of the Exact Match results page
- `400 Bad Request` — Invalid or missing `imageUrl`
- `503 Service Unavailable` — Google blocked the request (try adding proxies)
- `504 Gateway Timeout` — Request timed out

### `GET /health`

Health check. Returns pool size and status.

### `GET /`

Returns API info and usage example.

## Example

**Request:**
```bash
curl "https://your-api.com/google-lens?imageUrl=https://i.ebayimg.com/00/s/MTYwMFgxNjAw/z/BVcAAOSwS-9m4zOb/%24_57.JPG"
```

**Response:** Full HTML string of the Google Lens Exact Match page (typically 300K–1.7MB), containing links to pages where the exact image appears.
